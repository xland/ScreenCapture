# ScreenCapture

Мощный и легковесный инструмент для создания скриншотов в Windows, написанный на Qt/C++. Поддерживает межэкранный захват, скроллинг-скриншоты, рисование аннотаций и широкие возможности командной строки.

## Возможности

- **Межэкранный захват**, **скроллинг-скриншоты (длинные снимки)**, **поддержка высокого DPI**, **подсветка области окна**.
- **Пипетка** с горячими клавишами для копирования цветов RGB (`Ctrl+R`), HEX (`Ctrl+H`) и CMYK (`Ctrl+K`).
- **Инструменты рисования**: Заполненные/незаполненные эллипсы, идеальные круги (удерживайте `Shift`), прямоугольники, квадраты (удерживайте `Shift`), стрелки и нумерованные аннотации.
- **Линейные инструменты**: Кривые, прямые линии (удерживайте `Shift`), полупрозрачные линии, мозаика, ластик и текст.
- **Редактирование аннотаций** в любое время при наведении на них мышью.
- **Отмена** (`Ctrl+Z`) и **повтор** (`Ctrl+Y`).
- **Закрепление скриншотов** в новых окнах с возможностью масштабирования колесом мыши.
- **Сохранение в файл** (`Ctrl+S`) или **копирование в буфер обмена** (`Ctrl+C` или двойной щелчок).
- **Высокая производительность** с низким потреблением памяти и ЦП.
- **Портативность** - один исполняемый файл, установка не требуется, без внешних зависимостей.
- **Богатые параметры командной строки** для легкой интеграции в другие приложения.

## Командная строка

### Обзор параметров

Все параметры командной строки являются **необязательными**. Запуск без параметров показывает полный интерфейс скриншота.

### Базовые параметры

**--path (необязательно)**: Установить путь сохранения для скриншотов

```powershell
# Если путь - это каталог, имя файла будет сгенерировано автоматически
ScreenCapture.exe --path:"D:\\screenshots"

# Если путь включает имя файла, сохранение с указанным именем (перезапись, если существует)
# В настоящее время поддерживается только формат PNG
ScreenCapture.exe --path:"D:\\screenshots\\capture.png"
```

**--lang (необязательно)**: Установить язык приложения

```powershell
# Поддерживает упрощенный китайский (zhcn) и английский (en), по умолчанию zhcn
ScreenCapture.exe --lang:en
ScreenCapture.exe --lang:zhcn
```

**--comp (необязательно)**: Сжимать изображения при сохранении

```powershell
# Формат: --comp:качество,процент_масштаба
# Диапазон качества: от -1 до 9 (целое число)
#   0: Минимальное сжатие, самый большой файл
#   9: Максимальное сжатие, самый маленький файл
#  -1: Сжатие по умолчанию, среднее сжатие
# Диапазон процента масштаба: от 1 до 100 (целое число)
# Примечание: Параметр качества не влияет при сохранении в буфер обмена
ScreenCapture.exe --comp:6,60
```

### Параметры скриншота

**--cap (необязательно)**: Функция скриншота, взаимно исключает с --pin

**Скриншот области**:

```powershell
# Захват указанной области: x,y - координаты верхнего левого угла, width,height - размеры
# Сохранить в буфер обмена (параметр clipboard необязателен, сохраняет в файл если не указан)
ScreenCapture.exe --cap:area,10,10,500,600,clipboard

# Сохранить по указанному пути
ScreenCapture.exe --cap:area,10,10,500,600 --path:"D:\\screenshots"
```

**Скриншот полного экрана**:

```powershell
# Захватить весь экран и сохранить в буфер обмена (параметр clipboard необязателен)
ScreenCapture.exe --cap:fullscreen,clipboard

# Захватить весь экран и сохранить по указанному пути
ScreenCapture.exe --cap:fullscreen --path:"D:\\screenshots"
```

**Пользовательский скриншот**:

```powershell
# Пользователь перетаскивает мышь для определения области захвата, затем сохраняет в буфер обмена
ScreenCapture.exe --cap:custom,clipboard

# Пользователь перетаскивает мышь для определения области захвата, затем сохраняет по указанному пути
ScreenCapture.exe --cap:custom --path:"D:\\screenshots"
```

**Длинный скриншот**:

```powershell
# Скроллинг-скриншот только с четырьмя кнопками управления: pin, clipboard, save, close
# Также поддерживает настройку через параметр --tool
ScreenCapture.exe --cap:long
```

### Параметры закрепления

**--pin (необязательно)**: Функция закрепления, взаимно исключает с --cap

**Закрепить изображение из буфера обмена**:

```powershell
# Закрепить изображение из буфера обмена, x,y - координаты окна закрепления (необязательно)
# Координаты по умолчанию 100,100 если опущены
ScreenCapture.exe --pin:clipboard,100,100
ScreenCapture.exe --pin:clipboard
```

**Закрепить файл изображения**:

```powershell
# Закрепить указанный файл изображения, x,y - координаты окна закрепления (необязательно)
# Координаты по умолчанию 100,100 если опущены
ScreenCapture.exe --pin:file,"D:\\image.png",100,100
ScreenCapture.exe --pin:file,"D:\\image.png"
```

**Закрепить изображение области**:

```powershell
# Закрепить изображение области рабочего стола
# Формат: --pin:area,x,y,width,height,windowX,windowY
# Первые четыре параметра определяют область захвата, последние два - координаты окна закрепления (необязательно)
# Координаты окна по умолчанию 100,100 если опущены
ScreenCapture.exe --pin:area,100,100,500,600,800,10
ScreenCapture.exe --pin:area,100,100,500,600
```

### Параметры настройки интерфейса

**--tool (необязательно)**: Управление порядком, положением и видимостью кнопок панели инструментов

```powershell
# Доступные названия инструментов:
# Инструменты рисования: rect(прямоугольник), ellipse(эллипс), arrow(стрелка), number(нумерация), line(линия), text(текст), mosaic(мозаика), eraser(ластик)
# Инструменты действий: undo(отмена), redo(повтор), pin(закрепить), clipboard(буфер обмена), save(сохранить), close(закрыть)
# Разделитель: |
ScreenCapture.exe --tool:"rect,ellipse,arrow,number,line,text,mosaic,eraser,|,undo,redo,|,pin,clipboard,save,close"

# Пример упрощенной панели инструментов
ScreenCapture.exe --tool:"rect,arrow,|,clipboard,save,close"
```

### Примеры использования

```powershell
# Запуск без параметров - показывает полный интерфейс скриншота
ScreenCapture.exe

# Английский интерфейс с пользовательским путем сохранения
ScreenCapture.exe --lang:en --path:"D:\\screenshots"

# Быстрый скриншот в буфер обмена
ScreenCapture.exe --cap:custom,clipboard

# Сохранение скриншота высокого качества
ScreenCapture.exe --cap:area,0,0,1920,1080 --comp:-1,100 --path:"D:\\screenshot.png"

# Длинный скриншот с упрощенной панелью инструментов
ScreenCapture.exe --cap:long --tool:"pin,clipboard,save,close"
```

- Параметры `--cap` и `--pin` взаимно исключают друг друга, остальные параметры можно комбинировать.
- Параметры начинаются с `--`, не могут содержать пробелы и разделяются пробелами.
- Используйте соответствующие разделители путей (`\\` или `/`) в зависимости от вашего терминала при указании путей к файлам.

## Интеграция

Пример для Node.js или Electron (аналогично для Python):

```js
let spawn = require("child_process").spawn;
let child = spawn("./path/to/ScreenCapture.exe");
child.on("close", (code) => {
    /// Коды выхода:
    /// 0: не определено
    /// 1: выход по кнопке закрытия
    /// 2: выход по правой кнопке мыши
    /// 3: выход по клавише ESC
    /// 4: выход при копировании цвета RGB
    /// 5: выход при копировании цвета HEX
    /// 6: выход при копировании цвета CMYK
    /// 7: выход при копировании позиции мыши
    /// 8: выход при сохранении в файл
    /// 9: выход при сохранении в буфер обмена
    console.log("Код выхода:", code);
});
```

## Настройка горячих клавиш

### Вариант 1: AutoHotKey

- Скачайте и установите [AutoHotKey](https://www.autohotkey.com/).
- Создайте файл сценария `.ahk` в каталоге `%AppData%/Microsoft/Windows/Start Menu/Programs/Startup` со следующим содержимым:

```ahk
/*
^  :  Ctrl
!  :  Alt
+  :  Shift
#  :  Win

Пример 1: Ctrl+Alt+A 
^!A::Run "path/to/ScreenCapture.exe"

Пример 2: Win+Ctrl+Alt+P с параметром длинного скриншота
#^!P::Run "path/to/ScreenCapture.exe --cap:long"
*/

^!A::Run "path/to/ScreenCapture.exe"
```

- Дважды щелкните файл `.ahk`, затем вы можете запустить ScreenCapture.exe с помощью `Ctrl+Alt+A`.
- Этот сценарий будет автоматически запускаться при запуске Windows.

### Вариант 2: SC_Starter

[SC_Starter](https://github.com/Mikachu2333/sc_starter/) - это запускатель ScreenCapture, написанный на Rust.

- Поддерживает **автозапуск с Windows**
- **Встроенная программа скриншота**, дополнительная установка не требуется
- Автоматическая регистрация **глобальных горячих клавиш**
- Поддержка пользовательских путей сохранения по умолчанию
- Защита от случайного удаления
- Левый щелчок по системному трею для скриншота, правый щелчок для выхода
